LMS core overlay - tokenizeConvertCommand2 pipe guard
======================================================

Purpose
-------
Transcoder commands that already contain a shell pipe (e.g. Spotty spotty | lame)
must not receive LMS appending " & |" when $noPipe is unset. Append only when the
command has no internal "|"; otherwise playback stays silent.

Tracked file
------------
Host:    patches/lms/TranscodingHelper.pm (bind-mount over container path)
Mount:   ./patches/lms/TranscodingHelper.pm -> /lms/Slim/Player/TranscodingHelper.pm

Baseline when this overlay was created
--------------------------------------
Image:     lmscommunity/lyrionmusicserver:stable
Image ID:  sha256:c1099ea73717abd82d0e85aa4d7451717a36a3b2a4b9718ef139c8ce129f4508

Code change (conditional only)
------------------------------
In tokenizeConvertCommand2, guard the existing suffix append:

    if (!defined($noPipe) && $command !~ m!\|!) {
        $command .= (main::ISWINDOWS) ? "" : " &";
        $command .= " |";
    }

Optional: lame search path
--------------------------
The community image may lack /lms/Bin/x86_64-linux/lame (system lame is /usr/bin/lame).
After container recreate, Spotty MP3 may need:
  docker exec vbr-lms-1 ln -sf /usr/bin/lame /lms/Bin/x86_64-linux/lame
Or bake the symlink into a small entrypoint wrapper alongside this mount.

Bind-mount lame wrapper
----------------------
Host file patches/lms/lame-wrapper is mounted read-only at
/lms/Bin/x86_64-linux/lame so Spotty spt->mp3 can resolve [lame] without a
container-local symlink that disappears on recreate.
